# Einführung

Ein modales Fenster, das auf der Seite eingeblendet ist und vor dem Fortfahren eine Benutzerinteraktion erfordert.

# Hydrierung

**Stufe 1 – standardmäßig automatisch interaktiv.** Ein `Dialog` hat keinen sinnvollen statischen Fallback – Öffnen, Fokus-Trapping und ESC-Verarbeitung erfordern alle clientseitiges JavaScript – daher fungiert er standardmäßig als Insel. Übergeben Sie `interactive={false}`, um eine statische, inerte Dialog-Shell zu rendern, die kein Client-JS liefert.

| `interactive` prop | Ergebnis |
| --- | --- |
| omitted | Hydrates as an island (default) |
| `true` | Hydrates as an island |
| `false` | Static — no client JS |

Alle Interaktivitätsentscheidungen in der Bibliothek werden über den gemeinsam genutzten `shouldHydrate()`-Helper in `app/components/ui/island-utils.ts` weitergeleitet.

# Barrierefreiheit

Entspricht dem [Dialog (Modal) WAI-ARIA-Entwurfsmuster](https://www.w3.org/WAI/ARIA/apg/patterns/dialog-modal/). Bei Hydratisierung (`interactive` ist standardmäßig `true`) zeigt das Dialogfeld das folgende Verhalten an:

- **Der Fokus wird beim Öffnen in den Dialog verschoben** – auf `initialFocusEl()`, sonst auf das erste fokussierbare Element, sonst auf den Inhalt selbst.
- **Der Fokus bleibt beim Öffnen hängen** – `Tab` / `Shift+Tab` Zyklus nur innerhalb des Dialoginhalts. Verschachtelte Dialoge werden behandelt: Nur der oberste Dialog wird abgefangen und besitzt `Escape`.
- **`Escape` schließt** den Dialog (außer `closeOnEscape={false}`).
- **Hintergrund ist inert** – alles außerhalb des Dialogs (einschließlich eines übergeordneten Dialogs hinter einem verschachtelten) erhält `inert` + wird aus der Tab-Reihenfolge entfernt, während die Dialog-Unterstruktur interaktiv bleibt.
- **Body Scroll ist gesperrt**, solange mindestens ein Dialog geöffnet ist, und wird wiederhergestellt, wenn der letzte Dialog geschlossen wird.
- **Der Fokus kehrt beim Schließen zum Trigger zurück** (oder zu `finalFocusEl()`, falls angegeben).
- **Zugreifbarer Name** – abgeleitet von `title` (`aria-labelledby`) oder einem expliziten `aria-label`. Ein clientseitiger `console.warn` wird ausgegeben, wenn keiner vorhanden ist.
- **`role="alertdialog"`** – übergeben Sie `role="alertdialog"` für destruktive Bestätigungen und zeigen Sie `initialFocusEl` auf die Abbruch-/sichere Aktion, damit sie zuerst den Fokus erhält. Mit `closeOnInteractOutside={false}` kann der Dialog nur über einen Button oder `Escape` geschlossen werden.

# Verwendung

## Grundlegender Dialog

<DemoPreview>
  <Dialog
    trigger={<Button>Open Dialog</Button>}
    title="Confirm action"
    description="Are you sure you want to continue?"
    body="This action cannot be undone."
    cancel={<Button variant="outline">Cancel</Button>}
    confirm={<Button>Confirm</Button>}
  />
</DemoPreview>

```tsx
import { Dialog, Button } from "../components/ui";

export default function MyPage() {
  return (
    <Dialog
      trigger={<Button>Open Dialog</Button>}
      title="Confirm action"
      description="Are you sure you want to continue?"
      body="This action cannot be undone."
      cancel={<Button variant="outline">Cancel</Button>}
      confirm={<Button>Confirm</Button>}
    />
  );
}
```

## CMS Seiten-Builder

Diese Komponente ist als `dialog`-Block im [Page Builder](/docs/PageBuilder) (`content/pages/*.json`) verfügbar. `trigger` ist eine verschachtelte Blockliste (das CMS übermittelt immer ein Array):

```json
{
  "type": "dialog",
  "title": "Confirm action",
  "description": "Are you sure you want to continue?",
  "confirmText": "Confirm",
  "cancelText": "Cancel",
  "trigger": [{ "type": "button", "text": "Open Dialog" }]
}
```

# Eigenschaften

## Dialog

| Eigenschaft | Typ | Beschreibung |
| --- | --- | --- |
| `trigger` | `JSX.Element` | Element, das bei Aktivierung den Dialog öffnet. |

| `title` | \`string \ | JSX.Element\` | The dialog title. |
| `description` | \`string \ | JSX.Element\` | The dialog description. |
| `body` | \`string \ | JSX.Element\` | The main body content. |
| `footer` | \`string \ | JSX.Element\` | Custom footer content. |

| `cancel` | `JSX.Element` | Element, das als Auslöser zum Schließen (Abbrechen) gerendert wird. |
| `confirm` | `JSX.Element` | Als Aktionsauslöser gerendertes Element. |
| `closable` | `boolean` | Ob die Schließen-Schaltfläche angezeigt werden soll. Standard: `true`. |
| `interactive` | `boolean` | Aktivieren Sie clientseitige Flüssigkeitszufuhr für interaktives Verhalten. |
| `class` | `string` | Benutzerdefinierte CSS-Klassen für das Root-Element. |

| `role` | \`"dialog" \ | "alertdialog"\` | Dialog variant. Use `"alertdialog"` for destructive confirmations. Default: `"dialog"`. |

| `aria-label` | `string` | Zugänglicher Name, wenn kein `title` angegeben ist. |
| `closeOnEscape` | `boolean` | Schließen, wenn `Escape` gedrückt wird. Standard: `true`. |
| `closeOnInteractOutside` | `boolean` | Schließen, wenn auf den Hintergrund geklickt wird. Standard: `true`. |

| `initialFocusEl` | \`() => HTMLElement \ | null\` | Element to focus on open. Defaults to the first focusable. |
| `finalFocusEl` | \`() => HTMLElement \ | null\` | Element to focus on close. Defaults to the trigger. |

Zusätzliche Requisiten (z. B. `open`, `defaultOpen`, `onOpenChange`, `id`) werden an das zugrunde liegende Dialogprimitiv weitergeleitet.
