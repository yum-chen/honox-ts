# Einführung

Ein Panel, das vom Rand des Bildschirms eingeschoben wird, um Inhalte oder Aktionen anzuzeigen, ohne die aktuelle Seite zu verlassen.

# Verwendung

## Grundlegende Schublade

<DemoPreview>
  <Drawer
    trigger={<Button>Open Drawer</Button>}
    title="Settings"
    description="Adjust your preferences."
    body="Drawer body content goes here."
    cancel={<Button variant="outline">Close</Button>}
    confirm={<Button>Save</Button>}
  />
</DemoPreview>

```tsx
import { Drawer, Button } from "../components/ui";

export default function MyPage() {
  return (
    <Drawer
      trigger={<Button>Open Drawer</Button>}
      title="Settings"
      description="Adjust your preferences."
      body="Drawer body content goes here."
      cancel={<Button variant="outline">Close</Button>}
      confirm={<Button>Save</Button>}
    />
  );
}
```

## CMS Seiten-Builder

Diese Komponente ist als `drawer`-Block im [Page Builder](/docs/PageBuilder) (`content/pages/*.json`) verfügbar. `trigger` ist eine verschachtelte Blockliste (das CMS übermittelt immer ein Array):

```json
{
  "type": "drawer",
  "title": "Settings",
  "description": "Adjust your preferences.",
  "confirmText": "Save",
  "cancelText": "Close",
  "trigger": [{ "type": "button", "text": "Open Drawer" }]
}
```

# Eigenschaften

## Schublade

| Eigenschaft | Typ | Beschreibung |
| --- | --- | --- |
| `trigger` | `JSX.Element` | Element, das bei Aktivierung die Schublade öffnet. |

| `title` | \`string \ | JSX.Element\` | The drawer title. |
| `description` | \`string \ | JSX.Element\` | The drawer description. |
| `body` | \`string \ | JSX.Element\` | The main body content. |
| `footer` | \`string \ | JSX.Element\` | Custom footer content. |

| `cancel` | `JSX.Element` | Element, das als Auslöser zum Schließen (Abbrechen) gerendert wird. |
| `confirm` | `JSX.Element` | Als Aktionsauslöser gerendertes Element. |
| `closable` | `boolean` | Ob die Schließen-Schaltfläche angezeigt werden soll. Standard: `true`. |
| `interactive` | `boolean` | Aktivieren Sie clientseitige Flüssigkeitszufuhr für interaktives Verhalten. |
| `class` | `string` | Benutzerdefinierte CSS-Klassen für das Root-Element. |

Zusätzliche Requisiten (z. B. `open`, `defaultOpen`, `onOpenChange`, `id`) werden an das zugrunde liegende Schubladenprimitiv weitergeleitet.
